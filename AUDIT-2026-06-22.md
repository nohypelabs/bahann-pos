## CRITICAL FINDINGS (Fix Immediately)

### C1 — `Math.random()` di password reset token 🔴

**File:** `src/lib/email.ts:649-658`

```typescript
// SEKARANG (INSECURE):
const chars = 'ABCDEFGHIJKLMNOPQRSTUVWXYZabcdefghijklmnopqrstuvwxyz0123456789';
let token = '';
for (let i = 0; i < 64; i++) {
  token += chars.charAt(Math.floor(Math.random() * chars.length));
}

// HARUSNYA:
import { randomBytes } from 'crypto';
const token = randomBytes(32).toString('hex');
```

`Math.random()` bukan CSPRNG — attacker bisa predict token. Codebase sudah punya `crypto.randomBytes` di `refreshToken.ts`, tinggal pakai pattern yang sama.

### C2 — Rate limiter masih in-memory `Map` 🔴

**File:** `src/lib/security/rateLimiter.ts:11`

Masih `new Map()`. Di Vercel/serverless, setiap cold start reset. Attacker bisa bypass dengan hit instance berbeda. Login (5/15min) dan password reset (3/15min) sudah di-apply tapi **tidak efektif**.

**Fix:** Pakai `@upstash/redis` yang sudah ter-install.

### C3 — RLS enabled, zero policies 🔴

**File:** `supabase/migrations/016_enable_rls.sql`

RLS on di 14 tables, tapi **zero policies**. App pakai `supabaseAdmin` (service role) yang bypass RLS. Kalau service role key leak = **full database access**.

4 router baru (posDevices, transactionApprovals, expenses, outletGroups) ditambahkan tanpa abstraction layer sama sekali — semua logic inline di tRPC handler.

### M2 — Password reset token accept plaintext fallback

**File:** `src/infra/repositories/SupabasePasswordResetTokenRepository.ts:11`

```typescript
.in('token', [token, hashedToken]) // accepts BOTH
```

New tokens hashed, tapi query masih accept plaintext. Old plaintext tokens di DB masih valid. Plaintext fallback harus dihapus.

### M3 — No Content-Security-Policy header

`next.config.ts` set security headers tapi **tanpa CSP**. Untuk POS app yang handle transaksi finansial, CSP adalah layer pertahanan penting melawan XSS.

### M4 — `getAllUsers` leaks cross-tenant data

**File:** `src/server/routers/auth.ts:284-308`

`adminProcedure` + `findAll()` tanpa filter `tenant_id`. Admin tenant A bisa lihat users dari tenant B.

### M5 — `updateUserRole` no role enum validation

**File:** `src/server/routers/superAdmin.ts:180`

```typescript
role: z.string() // accepts ANY string
```

Harusnya `z.enum(['admin', 'cashier', 'manager', ...])`.

### M6 — No root `middleware.ts`

Edge-level route protection tidak ada. Semua auth rely di tRPC procedure guards. Tambah Next.js API route atau page tanpa guard = unprotected.

### M7 — `@supabase/auth-helpers-nextjs` deprecated

Masih dipakai di 3 file. `@supabase/ssr` belum di-install.

### M8 — PaymentModal zero a11y

**File:** `src/components/payment/PaymentModal.tsx`

Tanpa `role="dialog"`, `aria-modal`, focus trap, ESC handler. Ini modal pembayaran — salah satu UI paling kritis.
