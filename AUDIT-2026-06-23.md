## CRITICAL FINDINGS (Fix Immediately)

### C1 — Service Role Key Exposed to Client

**File:** `.env.local:3`

```
NEXT_PUBLIC_SUPABASE_SERVICE_ROLE_KEY=eyJhbGciOiJIUzI1NiIs...
```

`NEXT_PUBLIC_` prefix causes Next.js to bundle this into client-side JS. The service role key **bypasses all RLS**. Anyone inspecting the bundle gets full DB access.

**Fix:** Remove `NEXT_PUBLIC_` prefix. Use `SUPABASE_SERVICE_ROLE_KEY` (server-only).

### C2 — Public Registration Creates Admin Accounts

**File:** `src/server/routers/auth.ts:22-93`

`register` is `publicProcedure` with no CAPTCHA, no email verification, no invite system. Always creates `role: 'admin'`. Attacker can mass-create admin accounts with full tenant access.

**Fix:** Add rate limiting, email verification, or invite-only model.

### C3 — JWT Fallback Accepts 7-Day Tokens

**File:** `src/server/trpc.ts:39-68`, `src/lib/jwt.ts:13`

If Redis is down, system trusts JWT payload alone. Default `JWT_EXPIRES_IN = '7d'`. Stolen JWT valid for 7 days with no server-side revocation.

**Fix:** Change default to `'30m'`. Make Redis session validation mandatory.

### C4 — Rate Limiter Still In-Memory

**File:** `src/lib/security/rateLimiter.ts:11`

`new Map()` resets on every Vercel cold start. Login rate limit (5/15min) is theater. App already has `@upstash/redis` configured but rate limiter doesn't use it.

**Fix:** Rewrite to use Upstash Redis with atomic INCR + EXPIRE.

### C5 — Production Secrets in .env Files

**Files:** `.env`, `.env.local`

Real credentials on disk: Supabase keys, JWT secret, Resend API key, Sentry token, Redis token, Solana wallet. Not tracked in git, but any compromised dev machine = full production access.

**Fix:** Rotate all credentials. Use secrets manager (Vercel Env Vars, Doppler).

## MAJOR FINDINGS

### M1 — Password Reset Token Accepts Plaintext Fallback

**File:** `src/infra/repositories/SupabasePasswordResetTokenRepository.ts:11`

```typescript
.in('token', [token, hashedToken]) // accepts BOTH
```

Old plaintext tokens in DB still valid. Remove plaintext from query.

### M4 — `expenses.void` No Tenant Isolation

**File:** `src/server/routers/expenses.ts:196-199`

Uses legacy `role` check instead of RBAC. Does not verify expense belongs to user's tenant.

### M5 — `auth.refresh` Public, No Rate Limit

**File:** `src/server/routers/auth.ts:246-249`

`publicProcedure` with no rate limiting. Spam vector for DB exhaustion.

### M6 — `paymentRequests.uploadProof` No Input Size Limit

**File:** `src/server/routers/paymentRequests.ts:80-97`

`z.string()` with no `.max()` on base64 payload. Multi-GB string allocated in memory before size check.

### M7 — `updateSettings` Accepts Arbitrary Keys

**File:** `src/server/routers/superAdmin.ts:81-97`

`z.record(z.string(), z.string())` — no allowlist of valid setting keys.

### M8 — No Root `middleware.ts`

Edge-level route protection missing. All auth relies on tRPC procedure guards.

### M9 — RLS Deny-All, No Permissive Policies

**File:** `supabase/migrations/016_enable_rls.sql`

RLS on but zero policies. App uses `supabaseAdmin` (bypasses RLS). If service role key leaks = full DB access.

### M10 — No Content-Security-Policy Header

`next.config.ts` has security headers but no CSP. Critical for POS app handling financial transactions.
